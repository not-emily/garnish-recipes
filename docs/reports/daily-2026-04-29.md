## Decisions Made

`DECISIONS.md` not updated today — implementation-level calls, captured in commit messages and `PROJECT_PROGRESS.md`. Notable ones:

1. **Convention wins over user intent on display formatting.** Unit class drives the format (cup → fraction, gram → decimal). Considered but rejected adding a `quantity_format` JSONB hint to preserve typed form — convention is right ~95% of the time, and the hint adds storage surface for a marginal case.
2. **Parser is the spec.** Built `parseQuantity` and `formatQuantity` in TypeScript with an exhaustive Vitest suite. When the imported-recipe-parsing backlog item gets picked up later (Ruby), the same algorithm gets ported; the TS test cases become the contract both honor.
3. **Chip row visibility is unit-aware (not just focus-based).** `focused && unitClass(unit) === "fractional"` instead of just `focused`. Chips become a UI signal — "this unit makes sense as a fraction" — and avoid tempting users to make `355½ g` nonsense. Symmetric with the formatter rule.
4. **Auto-revert on invalid IngredientEditor qty blur (not validity-bubbling).** Considered the React-canonical `onValidityChange` callback pattern but the plumbing across IngredientRow → IngredientEditor → RecipeForm is ~30 lines for a personal-scale app. Auto-revert is ~5 lines, has no reorder-key edge cases, and keeps data + UI in sync without coordination. The bounce-back is the feedback.
5. **Pill chips (`rounded-full`) over box chips.** Matches the codebase's existing filter-chip convention (`RecipeFilterPanel.tsx`, `RecipeBrowser.tsx` both use `rounded-full`). Initial design used `rounded-md` which was inconsistent.
6. **4-column grid wrap, not flex-wrap.** Initial flex-wrap layout produced an awkward 7+1 wrap on phone widths (one orphan chip on row 2). Switched to `grid grid-cols-4 gap-1.5` for deterministic 4+4 wrap regardless of container width. Each chip fills 1/4 of the row.
7. **🌿 emoji for brand icon.** Literal name match, color alignment with `garnish-green`, distinctive at 16×16 favicon scale, platform-consistent rendering.
