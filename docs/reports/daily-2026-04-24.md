## Progress Today

**Phase 3 testing + fixes (ended up the largest investigation of the day):**
- Confirmed nav sizing, dead-zone, filter persistence, store categorization
- Fixed grocery-swipe vertical drift — Framer Motion's drag pipeline was writing a y value proportional to the x delta despite `drag="x"`. Literal `y: 0` in style is a static initial value, not a binding. Fix: real `useMotionValue(0)` for y, pinned via `onDrag` per-tick reset

## Decisions Made

`DECISIONS.md` hasn't been updated — today's deviations were captured in commit messages and `PROJECT_PROGRESS.md` instead. Worth noting:

1. **Kept the `MealPlanEntry` after_commit cook-stats trigger** despite phase-4.md's "remove incorrect trigger" instruction. Trigger is correct-but-incomplete (`date <= Date.current` guard handles future creates correctly); the gap is "future entry whose date passes," which the nightly job covers

4. **`touch-action: none` on swipeable items** (rejected) → kept `touch-pan-y`, fixed the underlying y-MotionValue issue instead. Avoids breaking list-scroll-from-an-item
